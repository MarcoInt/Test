Webserver Funktionalität

In den Folgenden 4 Abschnitten wird dir die Funktionalität eines Webservers erklärt

Statische Webseiten:

Statische Webseiten bestehen aus festen Inhalten, die sich nicht ändern, es sei denn, der Entwickler aktualisiert die Dateien manuell. Sie werden direkt vom Webserver an den Browser des Benutzers gesendet.

Dynamische Webseiten:

Dynamische Webseiten generieren Inhalte in Echtzeit basierend auf Benutzereingaben oder anderen Faktoren. Sie verwenden serverseitige Skriptsprachen, um Inhalte zu erstellen, bevor sie an den Browser gesendet werden.

Serverseitige Skriptsprachen:

Serverseitige Skriptsprachen wie PHP, Python und Node.js werden auf dem Server ausgeführt. Sie ermöglichen die Erstellung dynamischer Inhalte und die Interaktion mit Datenbanken.

Clientseitige Skriptsprachen:

Clientseitige Skriptsprachen wie JavaScript werden im Browser des Benutzers ausgeführt. Sie ermöglichen die Manipulation der Webseite nach dem Laden und die Interaktion mit dem Benutzer ohne erneutes Laden der Seite.

Das DOM:

Document Objekt Model

Was ist das DOM?

Das Document Object Model (DOM, engl. für Dokumenten-Objekt-Modell) ist eine Spezifikation einer Programmierschnittstelle, welche HTML- oder XML-Dokumente als eine Baumstruktur darstellt, in der jeder Knoten ein Objekt ist, welches einen Teil des Dokumentes repräsentiert, z. B. einen Absatz, eine Überschrift, ein Video oder etwa eine Tabellenzelle. Die Schnittstelle ist plattform- und programmiersprachenunabhängig und erlaubt damit standardisiert, die Struktur und das Layout eines Dokumentes zu verändern. Im Webbrowser bildet dies einen wichtigen Baustein für dynamische Webseiten.
